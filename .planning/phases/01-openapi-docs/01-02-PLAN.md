<task type="auto">
  <name>Task 1: Install swagger-ui-express and serve Swagger UI</name>
  <files>package.json, server/routes/openapi.ts, server/app.ts</files>
  <read_first>
    - server/routes/openapi.ts (created in Plan 01 — has GET /openapi.json, need to add Swagger UI routes)
    - server/app.ts (see how openapi router is mounted)
    - package.json (check existing deps)
  </read_first>
  <action>
1. Install: `npm install swagger-ui-express` and `npm install -D @types/swagger-ui-express`

2. Update `server/routes/openapi.ts` to add Swagger UI:
   - Import `swaggerUi` from `swagger-ui-express`
   - Import `generateMCPOpenAPISpec` from `../lib/mcp-to-openapi.js`
   - Generate the spec once at module load: `const spec = generateMCPOpenAPISpec(version, serverUrl)` where version comes from package.json and serverUrl defaults to `process.env.API_URL || 'https://api.visantlabs.com'`
   - Add `router.use('/docs', swaggerUi.serve, swaggerUi.setup(spec, swaggerOptions))` where swaggerOptions:
     ```
     {
       explorer: true,
       customSiteTitle: 'Visant API Docs',
       customCss: '.swagger-ui .topbar { display: none }',
       swaggerOptions: {
         persistAuthorization: true,
         tryItOutEnabled: true,
         filter: true,
         displayRequestDuration: true,
       }
     }
     ```
   - Keep the existing GET /openapi.json route

3. In `server/app.ts`, ensure the openapi router is mounted so `/api/docs` works (should already be `app.use('/api', openapiRouter)` from Plan 01).
  </action>
  <verify>
    <automated>node -e "const pkg = require('./package.json'); console.log('swagger-ui-express:', !!pkg.dependencies['swagger-ui-express'])"</automated>
  </verify>
  <acceptance_criteria>
    - `package.json` contains `swagger-ui-express` in dependencies
    - `server/routes/openapi.ts` contains `import swaggerUi from 'swagger-ui-express'` or equivalent
    - `server/routes/openapi.ts` contains `router.use('/docs', swaggerUi.serve`
    - swaggerOptions includes `persistAuthorization: true` (so API key stays across page reloads)
    - swaggerOptions includes `tryItOutEnabled: true`
    - The /openapi.json GET route still exists alongside the /docs route
  </acceptance_criteria>
  <done>swagger-ui-express installed and /api/docs route configured with try-it-out enabled</done>
