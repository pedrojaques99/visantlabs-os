<task type="auto">
  <name>Task 1: Create MCP-to-OpenAPI converter</name>
  <files>server/lib/mcp-to-openapi.ts</files>
  <read_first>
    - mcp-server/shared.ts (TOOLS array — each tool has name, description, inputSchema with JSON Schema properties)
    - server/lib/openapi-gen.ts (existing OpenAPI generator for legacy routes — reuse info/server/securitySchemes structure)
  </read_first>
  <action>
Create `server/lib/mcp-to-openapi.ts` that:

1. Imports `TOOLS` from `../../mcp-server/shared.js`
2. Exports `generateMCPOpenAPISpec(version: string, serverUrl: string): object`
3. For each tool in TOOLS, generates a POST path at `/api/mcp/tools/{tool.name}`:
   - operationId: tool.name
   - summary: first sentence of tool.description
   - description: full tool.description
   - tag: derived from tool name prefix (e.g. `generate_*` -> "Generation", `list_*`/`get_*`/`delete_*` -> "CRUD", `create_*`/`update_*` -> "Management", brand-related -> "Brand"). Use a simple mapping:
     - Names containing "brand" or "guideline" -> "Brand"
     - Names starting with "generate_" or "improve_" or "suggest_" or "extract_" -> "AI Generation"
     - Names containing "mockup" -> "Mockups"
     - Names containing "canvas" -> "Canvas"
     - Names containing "campaign" or "ad_" -> "Campaigns"
     - Names containing "creative" -> "Creative"
     - Default -> "Tools"
   - requestBody: `application/json` schema from tool.inputSchema (already JSON Schema, just reference it)
   - responses: 200 with generic MCP result schema `{ content: [{ type: "text", text: string }] }`, 401 unauthorized, 402 insufficient credits
   - security: `[{ apiKeyAuth: [] }]`
4. Also merge the legacy REST spec from `generateOpenAPISpec()` (import from openapi-gen.ts) so the full spec includes both MCP tools and REST endpoints
5. Set `openapi: "3.1.0"` in the output
6. Include securitySchemes for both bearerAuth (JWT) and apiKeyAuth (visant_sk_* format)
7. Include server entries for production (`https://api.visantlabs.com`) and local (`http://localhost:3001`)

The key design: TOOLS array is the single source of truth. No manual endpoint definitions for MCP tools.
  </action>
  <verify>
    <automated>npx tsx -e "import { generateMCPOpenAPISpec } from './server/lib/mcp-to-openapi.js'; const spec = generateMCPOpenAPISpec('1.0.0', 'http://localhost:3001'); const paths = Object.keys(spec.paths); console.log('Total paths:', paths.length); console.log('Has MCP tools:', paths.some(p => p.includes('mcp/tools'))); console.log('OpenAPI version:', spec.openapi);"</automated>
  </verify>
  <acceptance_criteria>
    - File `server/lib/mcp-to-openapi.ts` exists and exports `generateMCPOpenAPISpec`
    - Output contains `openapi: "3.1.0"`

    - Output paths include `/api/mcp/tools/document_extract` (last tool)
    - All 37 MCP tools appear as paths
    - Each MCP tool path has `post` method with requestBody and responses

    - securitySchemes includes both `bearerAuth` and `apiKeyAuth`
  </acceptance_criteria>
