  <files>server/routes/usage.ts</files>
  <read_first>server/routes/usage.ts, server/utils/usageTracking.ts</read_first>
  <action>
Add a new GET `/daily` route to the existing usage router in `server/routes/usage.ts`. This endpoint provides daily-aggregated usage data for charts.

Query params: `days` (default 30, max 90), `feature` (optional filter, same whitelist as /history).

MongoDB aggregation pipeline:
1. $match: userId + date range + optional feature filter
2. $group by { $dateToString: { format: "%Y-%m-%d", date: "$timestamp" } }, sum creditsDeducted, count calls
3. $sort by date ascending

Response shape:
```json
{
  "daily": [
    { "date": "2026-05-01", "calls": 42, "credits": 156 },
    { "date": "2026-05-02", "calls": 38, "credits": 120 }
  ],
  "period": { "from": "2026-04-19", "to": "2026-05-19", "days": 30 }
}
```

Apply same `usageHistoryLimiter` and `authenticate` middleware. Use same `connectToMongoDB`/`getDb` pattern as existing `/history` route. Validate `days` param is a number between 1-90. Use same feature whitelist (brandingmachine, mockupmachine, canvas).
  </action>
