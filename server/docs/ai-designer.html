AI Designer Agent

Complete reference for LLMs acting as design copilots. Covers onboarding, authentication, brand context, autonomous configuration, generation, moodboard extraction, and upscaling — via MCP, REST API, or Tool Call.

Step 0 — Onboarding & Authentication

Before any generation, the agent must ensure the user is authenticated. Ask in this order:

1
Ask for Visant API key.
"To use Visant as your design copilot, I need your API key. Get one at visantlabs.com/settings/api-keys (free account). Paste it here and I'll configure everything."
Key format: visant_sk_xxx — used as Authorization: Bearer visant_sk_xxx.
2
Verify the key works.
Call account-usage (MCP) or GET /api/account/usage (REST) with the key.
Returns: credits remaining, plan, limits. If 401 → key invalid, ask again.
3
Check for existing brand guidelines.
Call brand-guidelines-list. If empty → offer to create one or ingest from URL/text.
If multiple → ask which to use for this session.
The agent should never assume a key exists. Always validate before proceeding. If the user is new, guide them to register at visantlabs.com then return with their key.

MCP — full example

// 1. List guidelines
const list = await mcp('brand-guidelines-list', {});
const brand = list.guidelines[0]; // or let user choose

// 2. Get full context
const detail = await mcp('brand-guidelines-get', { id: brand.id, format: 'structured' });

// 3. Generate (brand context auto-injected by server)
const result = await mcp('mockup-generate', {
  promptText: 'Product hero shot. Dark studio. Brand logo at bottom.',
  brandGuidelineId: brand.id,
  model: 'gpt-image-2',
  aspectRatio: '1:1',
  resolution: '1K',
});

Moodboard Pipeline — Generate → Extract → Upscale

Generate a grid image (1 API call describing 9 sections), then extract individual cells and optionally upscale them.

Generate 3x3 moodboard
→
POST /api/moodboard/detect-grid
→
Extract cells (crop by bounding boxes)
→
POST /api/moodboard/upscale (per cell)
→
POST /api/moodboard/suggest (animation)

1. Detect grid cells

POST /api/moodboard/detect-grid
Authorization: Bearer visant_sk_xxx
{ "imageBase64": "{{base64 of the moodboard image}}" }

// Returns: { boxes: [{ id, x, y, width, height }, ...] }

2. Upscale a cell

POST /api/moodboard/upscale
Authorization: Bearer visant_sk_xxx
{ "imageBase64": "{{cropped cell base64}}", "size": "4K" }
// size: "1K" | "2K" | "4K"
// Returns: { upscaledBase64 }

3. Animation suggestions (optional)

POST /api/moodboard/suggest
Authorization: Bearer visant_sk_xxx
{ "images": [{ "id": "cell-1", "base64": "..." }, ...] }
// Returns: { suggestions: [{ id, animationPreset, veoPrrompt }] }
The moodboard prompt pattern: describe 9 sections inline as "Row 1: (1) ... (2) ... (3) ... Row 2: ..." — the model composes the grid. No stitching needed.

Logo Reference Rule

NEVER describe the logo in text. "A stacked geometric symbol with horizontal bars" → model will hallucinate. Always inject as reference image.
SituationEndpointHow
Logo needed in scenePOST /v1/images/editsmultipart, image[] = logo base64 blob
No logo neededPOST /v1/images/generationsJSON, string prompt
Via Visant APIPOST /api/mockups/generatePass brandGuidelineId — server injects logo automatically

When using the Visant API, logo injection is automatic — just pass brandGuidelineId. Priority: icon → primary → light → dark.

Auto-Injected Brand Context

When brandGuidelineId is passed, the server prepends to the prompt:

  • Logos — up to 2, as referenceImages (never described in text)
  • Media kit — up to 2 style references from media[]
  • Colors — full palette (hex + name + role)
  • Typography — font families and roles
  • Voice & tone — do's/don'ts, imagery guidelines
  • Strategy — positioning, archetypes, personas
  • Gradients & tokens — if defined in guideline

Prompt Rules

Rule✅ Do❌ Don't
